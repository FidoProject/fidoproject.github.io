The Fido Project

A libre C++ machine learning library for embedded electronics and robotics.

about fido

Real-time machine learning is the next step for mobile, integrated intelligent systems. However, most solutions are designed for laboratory-only use, and cannot be practically implemented in real-world resource-limited environments. Meet Fido, the machine learning library designed for integrated high-performance practical operation; Fido can run smoothly on devices from AVR micro-controllers to Raspberry Pi's.

Fido is loosely organized into three types of tools: systems, widgets, and utilities. Widgets are classes that represent useful machine learning devices: examples include neurons, layers, and neural networks. Utilities are what they say on the tin: examples include activation functions. Finally, systems are complete constructions of widgets, utilities, and other systems to form fully functional devices. This is what makes Fido truly powerful: a physical robot that learns from positive and negative reinforcement is only a line of code away.

getting started

First, download the latest release from Github. In the downloaded zip you will find amalgamated header and source files, Fido.h and Fido.cpp. Now, simply add the two files to your project and put the following at the top of wherever you're using Fido:

#include <Fido.h>

And you're done! For more information please visit the docs.

who is fido

Fido was created by programmers Joshua Gruenstein and Michael Truell from the Horace Mann School. The project was started as a universal robot control system using limited feedback, which can be read about here. Fido still continues to be developed in this direction, and the research is included in the library as systems. Fido is free software licensed under the GNU General Public License, or GPL 3.0.
